Was ist Git ?

Git und Github sind nicht das gleiche! Git ist ein VSC-Tool und GitHub ist eine Plattform die Git benutzt, es gibt auch andere zb. GitLab, GitBucket, BitBucket, Sourceforge und so weiter. Git wird überall benutzt und ist eins der wichtigsten Tools für Entwickler geworden. Dadurch können Entwickler von überall zusammenarbeiten. Git ist ein Versionverwaltungstool, das bei jedem comitten eine Art Schnappschuss von deinem Projekt macht. Hinterher hat man einen Stapel von Schnappschüssen.

Git-Befehle

git init

Initialisiert ein lokales git repository

git add .

Gibt an was alles in den commit rein soll

git commit -m “message”

Hinterlegt einen message/commit

git push

Schickt das lokale repository in das github repository

git log

Zeigt alle die bis dato erstellten commits ausführlich an

git log --oneline

Zeigt alle die bis dato erstellten commits in kurzform an

git branch “name”

Erstellt einen neuen Branch

git branch -a

Zeigt alle erstellten Branches an

git branch -d “name”

Löscht den ausgewählten Branch

git checkout

Wechseln zwischen den Branches

git pull

Lädt die aktuelle Version des Repository von GitHub runter

git clone

Kopiert ein fremdes Projekt von Github

git merge

Ist dafür da um zwei branches zusammenzuführen

Abkürzungen

A - Added (This is a new file that has been added to the repository)

M - Modified (An existing file has been changed)

D - Deleted (a file has been deleted)

Untracked (The file is new or has been changed but has not been added to the repository yet)

C - Conflict (There is a conflict in the file)

R - Renamed (The file has been renamed)

S - Submodule (In repository exists another subrepository)
